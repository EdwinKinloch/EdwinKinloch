Naming the Thing You Can't Name

I've been building a self-hosted multi-agent system on my own time for a while now — internally I call it Project Domain Expansion. It's not a product, and I'm not going to describe the stack or the architecture in any detail that would let someone reconstruct it. That's on purpose.

What I will say is that the hard part was never getting an agent to answer a question. It was getting it to remember the right things, forget the right things, and know the difference between the two without me babysitting every decision. Most demos skip this part because it isn't flashy. It's also where all the actual engineering lives.

The second hard part was building in stop points — places where the system has to wait for a human before it does something expensive or irreversible. It's tempting to build the fully autonomous version first and add guardrails later. I did it the other way around, and I'd do it that way again.

Fifteen years of production infrastructure taught me the same lesson twice: the system that fails loudly and safely beats the system that never fails, because the second one doesn't exist. That's true whether you're flipping a Blue/Green deployment or letting an agent make its own decisions overnight.
